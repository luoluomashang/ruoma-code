# humanizer-xiaoshuo

## Description
小说专用去 AI 痕迹工具。只保留对小说创作强相关的 7+1 条规则。
重点：「不是……而是……」句式清除（99% 筛选率）+ 稀疏排版合并 + AI 高频词替换 + 可选无用细节清除。

## Trigger
当用户提到「小说去AI」「小说humanize」「小说润色去AI」「humanizer小说」「去AI味」时使用。

## Architecture

### 调度模式：主进程 + Sub-agent

主进程（本 Skill 的调用者）负责：
1. 交互确认（章节范围、R8 开关）
2. 为每个章节启动独立 Sub-agent
3. 汇总结果报告

Sub-agent 负责：
- 接收单章文本 + 上下文（副文本/人设/主旨）
- 执行 prompt.md 中的 R1-R7（+ 可选 R8）规则
- 返回修改后文本 + 修改清单

### 并行执行

- 每个章节 = 一个独立 Sub-agent
- 多章节可并行启动（受系统并发限制）
- Sub-agent 之间无依赖，各自独立注入上下文

### 章节范围确认规则

当用户触发 Humanizer 时：

1. **用户已指定章节**（如"对第3章去AI味"）→ 直接执行指定章节
2. **用户未指定章节**（如"去AI味"）→ 询问用户：
   > 你想对哪些章节执行 Humanizer？
   > - 可以指定单章（如"第5章"）或范围（如"第3-7章"）
   > - 如果不指定，我会处理最近写完的那一章
3. **用户说"全部"或"所有章节"** → 列出章节清单确认后批量执行

### Sub-agent 上下文注入

每个 Sub-agent 接收（通过 prompt 参数传递）：
- 章节编号（必需，用于标注和汇总对应）
- 章节正文（必需，由主进程读取后内联，或传路径由 Sub-agent 读取）
- prompt.md 完整规则（必需）
- R8 开关状态（必需）
- 副文本路径（R8 开启时传递，Sub-agent 自行读取以下文件）：
  - 大纲/场景规划（本章主旨来源）
  - 人物设定/人设文档（角色核心特质）
  - 用户口头说明的本章重点（如有，内联传递）
- 前后章衔接段（如有，各 200 字，确保修改不破坏章节间过渡）

**副文本降级**：如用户开启 R8 但无法提供副文本，Sub-agent 基于正文自行推断主旨，并在修改清单中标注「主旨为推断，非明确来源」。

## Instructions
读取 prompt.md 获取完整规则，按上述架构执行调度。

## Input
小说章节文本（直接粘贴、指定文件路径、或指定章节范围）

## Output
每章：修改后的章节文本 + 修改清单（列出每处改动及原因）
汇总：各章修改统计（改动数量、R8 删除字数占比等）

## Notes
- 这是成稿后的清理工具，不是创作规范
- 与 xushikj 的 writing_rules.yaml / style_rules.yaml 不冲突——那些是创作时的规范，这个是后处理
- 可被 xushikj 或任何小说写作系统作为后置模块调用
- Sub-agent 之间完全独立，不共享上下文，适合并行
