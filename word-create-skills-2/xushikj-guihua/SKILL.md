# 叙事空间创作系统 - 规划模块

## 概述

执行宏观十二工作流的步骤 1-6 和步骤 11。从一句话概括开始，逐步扩展为完整的四页大纲，并在最后阶段完成书名与简介创作。每一步都严格贯彻六大法则，确保作品的商业潜力。

## 通用执行规则与卷次管理 (Volume Management)

### 1. 动态卷号变量 {V}
在执行涉及具体卷次大纲（如步骤 4、6）时：
- 必须先读取 `state.json -> planning_state.current_volume` 的值（设为 V）。
- 将所有提示词和输出路径中的 `{V}` 替换为实际数字。
- 例如：如果 `current_volume` 为 2，产出文件必须写入 `.xushikj/outline/volume_2_four_pages.md`。

### 2. 滚动大纲循环逻辑（开启下一卷）
当用户主动下达指令 **“开启下一卷”** 或 **“规划第 N 卷大纲”** 时，你必须执行以下动作：
1. 更新 `state.json`：将 `current_volume` + 1（或设为用户指定的 N）。
2. 将 `current_step` 回退重置为 4。
3. 读取当前最新的 `knowledge_base.json` 和上一卷的大纲结尾（提取钩子）。
4. 路由到步骤 4，向用户提问：“第 {V} 卷的规划已启动。承接上一卷的悬念，主角在这一卷的核心目标是什么？”

## 可用外部工具 (MCP Services)

你现在已连接 MCP 服务，拥有以下高级能力。你必须在特定场景下**主动调用**它们：

### 1. Websearch (网络搜索)
- **触发时机**：
  - 执行【步骤0：对标作品学习】时，主动搜索该作品的百度百科、番茄/起点书评、维基设定等，获取最准确的外部资料。
  - 执行【步骤2.5：世界观与力量体系】时，若需要真实的现实数据背书（如：狙击枪的有效射程、某种毒药的化学成分、风水学专业术语），必须搜索以保证“硬核爽感”。
- **调用要求**：搜索词必须精准，提取信息后必须提炼为符合小说设定的设定集。

### 2. Sequential-thinking (序列化深度思考)
- **触发时机**：
  - 执行【步骤4、6：大纲规划】时，若遇到多方势力博弈、复杂的反转或长线伏笔。
- **调用要求**：在输出最终大纲前，先调用此工具进行逻辑沙盘推演，确保“主角的破局手段在逻辑上完全成立”且“反派的算计没有低级漏洞”。

## 执行流程

### 核心原则

1. **每一步完成后必须等待用户确认**「满意」才能进入下一步
2. **每一步都要加载六大法则作为上下文**
3. **支持用户修改和迭代**，不满意可重新生成
4. **引导式交互**：每步开始前向用户提问，引导其输入核心信息

### 步骤1: 一句话概括

引导问题：

> 请用一句话告诉我，这是一个关于"谁"，在"什么情况下"，通过"什么方式"，达成了"什么结果"的故事？

要求：
- 不超过 25 个词
- 具有画面感，多用动词和名词
- 包含：小说类型、主角、核心任务
- 深度应用「极限铺垫原则」和「基因锚定」

产出保存到 `.xushikj/outline/one_sentence.md`。

### 步骤2: 一段式概括

将步骤1的一句话扩写为五句话的段落。

要求：
- 完整的三幕式结构
- 包含三次灾难性事件
- 清晰的道德前提
- 故事刺激、反复拉扯、最后有决定
- 应用「期待感管理系统」和「连锁震惊反应」

产出保存到 `.xushikj/outline/one_paragraph.md`。

### 步骤2.5: 世界观与力量体系设定

构建完整的世界观框架和力量运行规则。

要求：
- 明确世界的基本设定（地理、历史、文化等）
- 设定力量体系的运作机制和限制条件
- 确保世界观与故事发展紧密相关
- 应用「搭建舞台」和「设定规则」法则

产出保存到 `.xushikj/outline/worldview_and_system.md`。

### 步骤3: 一页人物介绍

严格遵循第四法则进行人物设计。

每个角色需包含：
- **名字**（含外号/绰号）
- **角色**（主角/女主/反派/配角）
- **价值观**
- **抱负**
- **目标**
- **矛盾**（内在冲突）
- **行为底层逻辑**（不妥协的执念）

女主设定需符合基因锚定法则。反派需有立场而非单纯善恶。

产出保存到 `.xushikj/outline/characters.md`。

### 步骤4: 一页大纲（可配置抽象层级）（第{V}卷）

将步骤2的每句话扩写成一个段落。

引导问题：

> 您希望这个大纲的详细程度是哪个级别？
> A. 故事级：仅包含几个核心转折点
> B. 幕级：分为清晰的三幕或五幕结构
> C. 序列级：每个大幕下包含若干连续情节序列
> D. 场景级：详细到每个具体场景的概要

> 写网文最忌讳开局就把几百万字的细节全写死。我们要用“滚动大纲”法！
> 现在，我们把视角收缩，只聚焦故事的**第一卷（通常是前10万字，或者第一个大高潮结束）**。
> 请告诉我，在这第一卷里，主角要解决的当务之急是什么？他将踩着哪个反派上位？第一卷的结尾，他将以怎样震撼的方式完成初次逆袭，并引出下一卷的悬念？

用户随时可在不同层级间切换（拉近/推远视角）。

产出保存到 `.xushikj/outline/volume_{V}_one_page.md`。

### 步骤5: 人物大纲

为每个核心角色撰写背景故事：

- 深挖过去经历、原生家庭
- 解释角色在小说中的行为与性格成因
- 设计人物弧光：经历重大事件后的成长与蜕变
- 主角的弧光必须与故事主线紧密结合

产出保存到 `.xushikj/outline/character_arcs.md`。

### 步骤6: 四页大纲（第{V}卷详细展开）

将步骤4的一页大纲扩展为第{V}卷的完整大纲，每段扩展为一页。

引导问题：
> 现在我们要把第{V}卷的骨架，填满血肉，扩展为详细的四页大纲。这将是我们马上开始逐章写正文的直接图纸。
> 我会将第{V}卷拆分成若干个连续的情节序列（Sequence），每个序列包含几个关键的爽点和冲突。您准备好详细打磨这些情节了吗？

要求：
- 全面应用六大法则构建详细情节
- 支持可配置抽象层级微调
- 标注每个情节点对应的法则应用
- 标注信息差释放的层级和节奏
- 标注钩子九连环的布局

产出保存到 `.xushikj/outline/volume_{V}_four_pages`。

### 步骤11: 书名与简介创作（特殊逻辑）

本步骤通常在步骤10（逐章创作）完成后执行，但也可在步骤6完成后提前执行。

**角色切换**：此步骤中 AI 扮演「顶级营销编辑」。

**书名创作**：
- 提供 3-5 个候选书名
- 书名需通俗易懂，紧贴核心题材和最大卖点
- 参考番茄热榜风格

**简介创作**（黄金结构模板）：
- **开篇定调**：一句话点明核心梗或最大反转
- **核心梗概**：简明扼要介绍主线和人设亮点，不剧透关键情节
- **钩子收尾**：开放式问题或高能预告，激发读者好奇心

产出保存到 `.xushikj/outline/title_intro.md`。

## 引导与确认模式

每一步遵循统一的交互模式：

```
1. 引导提问 → 收集用户输入
2. 加载六大法则 + 前置步骤产出
3. 生成内容
4. 展示给用户
5. 询问：满意 / 修改 / 重新生成
6. 用户确认「满意」→ 保存 → 更新 state.json → 进入下一步
```

## 注意事项

- 步骤1-6为严格顺序执行，不可跳步（除非用户使用灵活启动模式并提供前置产出）
- 步骤11可在步骤6后或步骤10后任意时刻触发
- 所有产出必须深度贯彻六大法则，不是走过场
- 如果用户已执行步骤0，所有输出的风格必须与对标风格报告一致

## 完成后

步骤6完成后，告知用户规划阶段完成，询问是否进入下一步：知识库模块（xushikj-zhishiku）。

步骤11完成后，告知用户书名简介已创作完毕，可供发布使用。
